# Phase 01: Internet Fundamentals

Welcome to Phase 1. Before writing any backend code, it is critical to understand the infrastructure that allows computers to communicate globally. This phase breaks down the Internet from first principles, covering everything from domain names and IP addresses to the TCP/IP stack and the HTTP request lifecycle.

---

## 🎯 Learning Objectives

By the end of this phase, you will be able to:
- Explain exactly what happens when a user types a URL into their browser.
- Understand how DNS translates human-readable domains into machine-readable IPs.
- Differentiate between TCP and UDP, and know when to use each.
- Trace the lifecycle of an HTTP request and understand status codes, headers, and methods.
- Distinguish between client-side and server-side responsibilities in a network architecture.

---

## 📚 Topics Covered

1. **The Internet Infrastructure**: Routers, ISPs, and the decentralized mesh.
2. **DNS & Addressing**: Domain Names, IPv4/IPv6, and resolution flows.
3. **Transport Layer**: The TCP 3-way handshake vs. UDP connectionless delivery.
4. **HTTP Protocol**: Requests, responses, headers, cookies, and sessions.
5. **Security Basics**: HTTPS and the TLS handshake.

---

## 🚦 Prerequisites

- No prior backend programming experience required.
- Basic familiarity with using a web browser.
