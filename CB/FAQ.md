## General Questions

### What is this repository?

Complete Backend Engineering is an open-source handbook that teaches backend engineering from first principles to production-ready systems.

Unlike traditional tutorials, this repository focuses on understanding **why** things work before learning **how** to implement them.

---

### Who is this repository for?

This repository is designed for:

- Students
- Self-taught developers
- Backend beginners
- Full Stack Developers
- Software Engineering Interns
- Backend Engineers
- Product Engineers

---

### Is this a course?

No.

This repository is a documentation-based learning resource.

Think of it as an open-source backend engineering handbook rather than a video course.

---

### Is the content free?

Yes.

Everything in this repository is completely free and open source.

---

### Can I use this repository for interview preparation?

Absolutely.

Every module includes:

- Theory
- Production concepts
- Interview questions
- Assignments
- Mini projects
- Best practices

## Learning Path

### Where should I start?

Start from:

```text
01 Internet Fundamentals
```

Do not skip topics.

Many advanced backend concepts depend on networking fundamentals.

---

### Can I jump directly to JWT or Docker?

You can.

However, it is highly recommended to follow the learning roadmap in order.

Backend engineering builds layer by layer.

---

### How long will it take to finish?

This depends on your pace.

Suggested schedule:

- 1 Topic per day
- 5–10 hours per week
- Around 2–3 months for a solid understanding

Remember:

Learning backend engineering is a marathon, not a sprint.

---

### Do I need to finish one module before moving to the next?

Yes.

Each module builds on concepts introduced earlier.

---

## Repository Structure

### Why are there so many small markdown files?

Each topic has its own documentation page.

This makes:

- navigation easier
- searching easier
- future updates simpler
- contributions more manageable

---

### Why are there separate phases?

The repository follows the same order as real backend systems.

```text
Internet
↓
HTTP
↓
REST APIs
↓
Authentication
↓
Databases
↓
Caching
↓
Async Systems
↓
Real-Time Systems
↓
Deployment
```

Understanding this order makes learning much easier.

---

### Where can I find diagrams?

All diagrams are stored inside:

```text
assets/
```

or inside each module's dedicated `assets/` folder.

### Do I need to know Java?

No.

Basic programming knowledge is enough to begin.

Yes.

## Career & Interviews

### Will this help me get backend internships?

It is designed to prepare you for:

- Backend internships
- Product engineering roles
- Startup engineering roles
- Backend interviews

However, interview success also depends on:

- DSA
- Projects
- Communication
- System Design

---

### Does this include System Design?

Not initially.

System Design will be added as an advanced module after the backend fundamentals.

---

### Is this enough for FAANG?

This repository provides strong backend fundamentals.

For FAANG-level preparation, you should additionally practice:

- DSA
- Low-Level Design
- High-Level System Design
- Mock Interviews

## Technical Questions

### Why isn't this focused on Express, Spring Boot, or Django?

Frameworks change.

Fundamentals remain.

This repository teaches concepts that apply across all backend technologies.

---

### Why are production examples included?

Understanding how companies solve real-world problems helps bridge the gap between tutorials and production systems.

---

### Why are there so many diagrams?

Backend engineering is easier to understand visually.

Diagrams help explain:

- request flows
- networking
- architecture
- databases
- distributed systems
