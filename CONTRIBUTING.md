# Contribution Guidelines

Thank you for your interest in contributing to the Backend Engineering Handbook!

## Good First Issue
Check out issues labeled good first issue to get started.

## Issue Templates
Please use the provided issue templates for bug reports and feature requests.

## Pull Request Template
Please use the PR template and ensure your changes pass all checks.

## Folder Naming Rules
Use lowercase folders with hyphens (e.g., phase-01-internet-fundamentals).

## Markdown Rules
- Use GitHub callouts (> [!NOTE], etc.)
- Use code blocks
- Use tables
- Use collapsible sections
- Use anchor links and relative links
- Follow proper markdown formatting
